# 诉讼准备 Skill CN

面向中国民事诉讼场景的结构化分析 Skill。

它用于帮助用户在起诉、协商或咨询律师前，先把案件整理成清楚的诉讼结构：

## 在线体验 / 导入使用

你可以下载本仓库 ZIP 后导入支持 Skill / Agent 的工具中使用，例如 QClaw、LobeChat、Dify、Coze 等。

如使用后仍对案由、请求权基础、证据结构或诉讼策略不放心，可联系作者律师进行人工复核。

**案由识别 → 请求权基础 → 构成要件事实 → 证据清单 → 诉讼路径建议**

---

## 核心能力

### 1. 案由识别

根据用户陈述，初步判断案件可能属于什么民事案由，并区分：

- 推荐案由
- 备选案由
- 不建议案由
- 仍需补充的事实

### 2. 请求权基础分析

本 Skill 不只判断“是什么案由”，还会进一步分析：

- 用户的诉讼请求依据什么请求权基础；
- 哪个请求权基础适合作为主位路径；
- 哪些路径可以作为备选；
- 哪些路径不建议主张。

### 3. 构成要件事实拆解

针对主请求权基础，进一步拆解：

- 需要证明什么事实；
- 当前证据是否足够；
- 还缺什么关键证据；
- 谁承担举证责任；
- 对方可能如何抗辩。

### 4. 证据清单生成

将证据分为：

- 强证据
- 补强证据
- 缺失关键证据
- 高风险证据

### 5. 诉讼路径建议

根据事实和证据，给出：

- 是否适合立即起诉；
- 是否应先补证据；
- 是否建议先协商或调解；
- 是否需要律师进一步介入。

## 如何使用

### 方式一：复制 Skill

打开：

`skill/SKILL.md`

将其中内容复制到支持自定义系统提示词的 AI 工具中，例如：

- 腾讯龙虾 / AI Agent 工具
- LobeChat
- ChatGPT 自定义 GPT
- Dify
- Coze
- FastGPT

然后将：

`data/case-database.md`

作为知识库内容导入或复制给 AI 使用。

---

### 方式二：按模板输入案件事实

打开：

`templates/case-intake-template.md`

按照模板填写案件事实，再让 AI 进行分析。

---

### 方式三：输出标准分析报告

AI 应当参考：

`templates/litigation-report-template.md`

输出结构化的诉讼准备分析报告。
